## 滑动窗口

了解单调队列的同学肯定都做过滑动窗口，那么很自然的单调队列可以优化形式如下的动态规划：

$$dp_i = \min_{i - k \le j <i} \{ dp_j + cost_1(i) + cost_2(j) \}$$

具体实现就是将$dp_j + cost_j$ 作为滑动窗口中元素的值。

回到滑动窗口问题本身，我们发现滑动窗口的算法描述如下（本文中**仅讨论求滑动窗口中最小值**）：

1) 新建一个空的双端队列
2) 每当我们需要在滑动窗口中添加一个新元素时，**重复**比较其与当前双端队列队尾的大小，只要新元素更小，就删除此时双端队列的队尾
3) 每当我们需要在滑动窗口中删除一个元素时，因为删除的元素要么不在双端队列中，要么为双端队列中的队头的元素，所以仅当其为双端队列中的队头的元素时将其删除。
4) 当我们需要输出答案时，输出**队头的元素**。

我们很多同学对单调队列的理解就到此为止了，但是随着oi考试的难度升级，以上的理解往往在实战中就显得捉襟见肘，我们再次去考察我们解决滑动窗口问题时的一些**隐藏的性质**。

## 隐藏的性质

1) 性质一： 我们输出的答案对应元素的位置总是单调递增的
我们不难发现这一点：因为每次我们输出的答案总是队头的元素，而队头的元素在上文的**操作2**和**操作3**中只会向后移动。

1) 性质二： 我们可以设计一个**错误的算法**（不含**步骤2**），该错误算法在输入为一个**单调递增序列**时答案仍然正确。
因为如果输入为一个单调递增序列，那么**操作2**的比较操作不会满足条件。

有了这两个隐藏的性质我们发现滑动窗口其实可以**解决的是这样一类问题**：

1) 决策具有单调性（即每次转移选择的上个状态下标会单调递增）
2) 可选择的状态之间**大部分时候**没有单调性 (即$dp_j + cost_2(j)$ 不是关于$j$的单调函数）
3) 可选择的状态范围$[L(i), R(i))$ 满足L，R函数都**单调递增**

为什么可选择的状态之间**大部分时候**没有单调性，因为如果有单调性往往题目直接贪心就对了。

也就是说我们之前的限制条件过于严格了，其实单调队列此时优化的问题可以写作：

$$dp_i = \min_{L(i) \le j <R(i) } \{ dp_j + cost_1(i) + cost_2(j) \}$$

也就是说我们只需要将 $dp_j + cost_2(j)$ 作为元素插入单调队列中即可，每次决策取优先队列**队头**

## 第二类问题

我们上面描述了一个可以很好地使用一个单调队列解决的问题，但是有的时候，$L(i)$, $R(i)$两个函数不符合我们的要求，或者，dp的方程不符合上述的要求我们就要考虑单调队列的其他运用场景。

### 例题

[problem:洛谷-P4954](https://www.luogu.com.cn/problem/P4954)

我根据倒着做（将数组倒序输入），再利用贪心得出我们只需要保证每个状态稻草堆最下层宽度最小，$dp_i$表示前i个稻草堆最下层最小宽度：

$$dp_i = \min_{j <i \&\& S_i - S_j  > dp_j } \{ S_i - S_j  \}$$

我们发现 $S_i - S_j  > dp_j$ 这个条件成反而为了制约我们做题的关键。

### 例题问题抽象

两种操作：
1) 添加点$(x, y)$
2) 给定$h$, 查询满足$x \le h$ 中最小的$y$

显然可以使用线段树解决问题，如果满足操作1的$y$和操作2的$h$都单调递增，则可以使用单调队列。

我们考虑如果操作1的$x$也单调递增可以设计以下算法

1) 对于操作1，将$(x,y)$插入队列
2) 对于操作2，进行循环：如果队首的$(x,y)$中的$x$比h小将其出队，将$ans \leftarrow y$，

但$x$可能并非单调递增，那么我们考虑对于操作1，如果新插入的点比队尾的点$x$更小，则删除队尾的元素。

对于这个问题，我们其实是在用单调队列**维护即将进入合法范围的元素**，而之前的问题是在**维护合法范围内，即将成为最优转移点的元素**。

## WHAT‘S MORE?

斜率优化问题为什么有的时候可以用单调队列帮助解决?

类似这个问题：

[problem:洛谷-P2900](https://www.luogu.com.cn/problem/P2900)

我们后面需要解决的问题可以写作：

$$dp_i = \min_{L(i) \le j <R(i) } \{ dp_j + cost(i, j) \}$$

之前我们的$cost(i,j) = cost_1(i) + cost_2(j)$

现在$cost(i,j) = cost_1(i) + cost_2(j) + cost_3(i) \times cost_4(j)$

仍然要满足的性质：

1) 决策具有单调性（即每次转移选择的上个状态下标会单调递增）
2) 可选择的状态范围$[L(i), R(i))$ 满足L，R函数都**单调递增**

不过是单调队列里面的元素不能是简单的一个$dp_j + cost_2(j)$，而是一个点$(dp_j + cost_2(j), cost_4(j) )$， 两个点如何比较大小？这个需要利用一些凸包的知识，我们在以后数学的专题再去详细解答，其做法有点类似上面两个问题的结合。

对于这个问题，我们其实也是在用单调队列**维护在合法范围，即将成为最优转移点的元素**。

## 万变不离其宗

我们可以认为单调队列中元素都是二元组，其主要的功能是去除一堆元素中有**偏序关系（不可能影响最终答案）** 的部分（从队尾删除的都是满足该**偏序关系**不会在之后影响答案的部分，这样能保证队列中的元素在**两个维度都有单调性**），而从队头出队的，表示的是其**从某一时刻起**不再是最优的决策点（曾经可能是最优的决策点）。

单调队列可以认为是**依赖一个维度的单调性，从而在另一个维度上创造单调性的工具**。在单调队列之后又会利用新创造的单调性解决这些问题。**可以带着这个思路再去回顾本文中的问题。**
